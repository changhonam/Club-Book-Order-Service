# PRD (제품 요구사항 정의서)

## 1.1. 제품 개요
본 시스템은 독서동호회 회원들이 Yes24 도서 링크를 통해 간편하게 구매할 도서를 신청하고, 본인 부담금을 확인하며, 관리자가 이를 일괄적으로 수집하여 구매를 진행할 수 있도록 지원하는 사내/동호회용 웹 서비스이다.

## 1.2. 타겟 사용자
- **일반 회원**: 매월 읽을 도서를 신청하고, 자신의 결제 금액을 확인하는 사용자.
- **관리자 (구매 진행자)**: 회원 명부를 관리하고, 취합된 도서 신청 목록을 바탕으로 실제 도서 구매를 진행하며, 월별 정산을 관리하는 사용자.

## 1.3. 핵심 기능 요구사항

### 1.3.1. 회원 인증 및 권한

**이름 + PIN 로그인**: 회원 이름과 4자리 PIN 입력으로 서비스에 접근.

**보안 로직 (명부 기반 인증)**:
- 입력된 이름이 DB(Google Sheets - Members 시트)에 등록된 이름인지 검증.
- 이름이 존재하면 PIN(4자리 숫자)을 대조하여 인증. 신규 회원의 기본 PIN은 "0000".
- **제약사항**: 동명이인은 '김철수A', '김철수B' 형태로 명부에 사전 등록되어 고유성을 보장한다고 가정함.
- 이름과 PIN이 일치하면 로그인 성공. 미등록 이름일 경우 "관리자에게 등록을 요청하세요" 메시지 출력 및 접근 차단.
- 회원은 프로필 페이지에서 본인의 PIN을 변경할 수 있음. 관리자는 회원의 PIN을 "0000"으로 초기화 가능.

**회비 납부 확인**:
- 회원별 회비 납부 상태(fee_paid)를 Members 시트에서 관리.
- 회비 미납(fee_paid=false) 회원은 도서 신청이 불가하며, "회비 납부 후 도서 신청이 가능합니다" 안내.
- 사이드바에 회비 납부 상태 표시.

**관리자 인증**:
- 관리자 이름은 환경변수(`st.secrets["admin"]["name"]`)로 지정.
- 관리자 이름으로 로그인 시 별도 비밀번호 입력 폼을 노출하며, 비밀번호는 `st.secrets["admin"]["password"]`와 대조하여 인증.
- 일반 회원 뷰와 관리자 뷰를 분리하여 제공.

### 1.3.2. 도서 구매 신청 (일반 회원)

**URL 기반 도서 정보 자동 완성**: Yes24 도서 페이지 URL 입력 후 [정보 불러오기] 버튼 클릭 시 동작.

**모바일 URL 변환**: 사용자가 모바일 링크(`m.yes24.com/Goods/...`)를 입력한 경우, 스크래퍼가 이를 PC 링크(`www.yes24.com/Product/Goods/...`)로 자동 변환하여 처리.

**추출 항목**: 책 제목, 저자, 출판사, 판매가, ISBN-13, 구매 가능 여부, 도서 형태(eBook 여부).

**가격 파싱 기준**:
- Yes24 상품 페이지에서 **'판매가'로 표기된 가격**을 기준으로 파싱한다.
- '크레마머니 최대혜택가' 등 별도 할인 가격은 파싱 대상에서 제외한다.
- 판매가가 없는 경우(예: 무료 도서) 정가를 사용한다.

**구매 가능 여부 및 예외 검증**:
- 도서 페이지 내 '바로구매' 또는 '카트에 넣기' 활성화된 일반 종이책 → [구매 가능]
- '품절', '절판' 상태 → [구매 불가능] (신청 차단)
- eBook(크레마) 도서 → [구매 불가능] (신청 차단)
- **세트/묶음 상품**: 별도 구분 없이 일반 도서와 동일하게 처리. URL 단위 1건 주문.
- **스크래핑 실패**: 안내 메시지 노출 후 신청 차단. 일반 회원 수동 입력 불허, 관리자 대리 신청으로 처리.

**신청 관리**:
- 구매 가능한 도서에 한해 [구매 신청] 버튼으로 해당 월 목록에 등록.
- **동일 도서 중복 신청 허용** (같은 책 여러 건 신청 가능).
- 본인이 신청한 도서의 취소 및 수정(삭제 후 재등록) 가능.

### 1.3.3. 현황 조회 및 정산 (일반 회원)

**내 정보 조회 (대시보드)**:
- 월 선택 드롭다운 제공. 최근 12개월 중 주문 데이터가 존재하는 월만 표시하며, 현재 접수 월은 데이터 유무와 관계없이 항상 포함. 기본값은 현재 신청 접수 중인 월.
- 선택한 월의 본인 신청 도서 목록 및 정산 결과 조회 가능.
- **과거 월은 조회만 가능** (신청/취소 불가).

**월 누적 자동 정산 로직**:
- **"해당 회원의 해당 월 총 신청 누적 금액"** 기준 계산.
- 동호회 지원금 = 총액의 50% (최대 30,000원 한도), **원 단위 내림(floor)** 처리.
- 본인 납부 금액 = 이달 신청 총액 - floor(Min(이달 신청 총액 / 2, 30000원))

### 1.3.3.1. 본인 부담금 입금 확인

**입금 완료 처리**:
- 본인 부담금이 발생한 경우, 입금 안내(계좌 정보)와 함께 [입금 완료] 버튼을 제공.
- 회원이 [입금 완료] 버튼을 클릭하면 해당 월의 입금 상태가 "완료"로 기록됨.
- 입금 완료 시 안내 메시지가 "입금이 확인되었습니다."로 변경됨.
- 관리자 현황 모니터링에서 회원별 총액·지원금·본인부담금·입금 상태(완료/대기) 확인 가능.

### 1.3.4. 관리자 기능 (어드민)

**회원 관리**: 조회(이름, PIN 마스킹, 회비 납부 상태), 추가(이름 입력 폼, 기본 PIN=0000), 삭제(확인 메시지 포함), PIN 초기화(0000). 삭제 시 기존 Orders 데이터 보존.

**회비 관리**:
- 빠른 납부 체크: 회원 이름 입력 → 납부 대기 리스트에 추가 → '일괄 납부 처리' 버튼으로 한 번에 처리. 미등록/이미 납부/중복 입력 시 안내 메시지 표시. 리스트에서 개별 제거 가능.
- 납부 현황 대시보드: 전체/납부/미납 인원 및 목록 표시.
- 전체 회비 초기화: 모든 회원의 납부 상태를 미납으로 일괄 초기화.

**관리자 대리 신청**:
- 특정 회원 이름 선택하여 해당 회원 명의로 도서 신청 가능.
- **수동 입력 폼**(제목, 저자, 판매가 직접 입력) 제공하여 스크래핑 불가 도서도 등록 가능.

**신청 월 관리**: 관리자가 "현재 신청 접수 월" 설정. Config 시트에 저장.

**신청 마감 관리**:
- "신청 마감" 토글 버튼으로 마감/해제.
- **자동 마감 예약**: 특정 날짜+시간 지정 시 해당 시점에 자동 마감 (접속 시 체크 방식).
- 마감 상태에서도 관리자는 대리 신청 가능.

**현황 모니터링**: 특정 월의 전체 회원 신청 도서 목록과 총 필요 예산 대시보드.

**카트에 담기용 URL 일괄 복사**: 선택한 월의 신청 도서 Yes24 URL을 Chrome Extension("Yes24 장바구니 일괄 담기") 입력 포맷(`https://www.yes24.com/Product/Goods/{상품번호}`, 신청 건수만큼 줄바꿈 반복)으로 출력. `st.code` 내장 복사 버튼 제공. 파싱 불가 URL은 경고 표시 후 제외.

**월별 내역 엑셀 Export**: .xlsx 또는 .csv 파일로 다운로드.

**월간 데이터 관리 (안전 삭제)**: 특정 월 선택 삭제, 확인(Confirmation) 메시지 필수.
