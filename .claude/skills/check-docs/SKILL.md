# 문서 동기화 체커

코드 변경 사항을 분석하여 프로젝트 문서가 최신 상태인지 확인하고, 필요하면 직접 갱신한다.

## Step 1: 변경 사항 파악

인자가 주어지면 해당 ref 기준으로, 없으면 아래 순서로 변경 사항을 수집한다:

1. `git diff --staged` — staged 변경 확인
2. `git diff` — unstaged 변경 확인
3. `git status --porcelain` — untracked 파일(??로 시작하는 항목) 확인하여 새 파일 내용도 수집
4. 1, 2, 3 모두 비어있으면 `git diff HEAD~1` — 최근 커밋 변경 확인

인자가 있을 경우:
```bash
git diff $ARGUMENTS
```

변경된 파일 목록과 diff 내용을 모두 파악한다.

## Step 2: 문서 수집 및 비교

1. **문서 파일 수집**: Glob으로 프로젝트 내 문서성 파일을 탐색한다.
   - `**/*.md`

2. **문서 일괄 읽기**: 수집된 문서를 모두 읽는다.

3. **diff와 대조**: 읽은 문서 내용과 Step 1의 변경 사항을 비교하여 불일치를 찾는다.
   - 문서에 기술된 파일 구조가 실제와 일치하는지
   - 문서에 기술된 환경 변수가 코드에서 사용하는 것과 일치하는지
   - 문서에 기술된 처리 흐름이 현재 코드 로직과 일치하는지
   - 문서에 기술된 함수명, 클래스명, 설정값이 정확한지

## Step 3: 문서 갱신

불일치가 발견되면:
1. 구체적으로 어떤 부분이 불일치하는지 보고
2. 문서를 직접 수정하여 코드와 동기화

불일치가 없으면:
- "문서가 최신 상태입니다." 라고 보고

## 출력 형식

```
### 문서 동기화 결과

| 문서 | 상태 | 변경 내용 |
|---|---|---|
| (문서 경로) | ✅ 최신 / ❌ 갱신함 | (변경 설명) |
| ... | ... | ... |
```
